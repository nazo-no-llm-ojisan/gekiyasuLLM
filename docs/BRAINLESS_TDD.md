# 脳レスTDD開発

考えない。賢くしない。**赤 → 緑 → コミット** だけ。

名前の意味: 設計の深掘りや大きなリファクタをその場でやらない。  
「次に通すべきテスト1本」以外は脳を使わない。

---

## ルール（破らない）

1. **本番コードを先に書かない。** 失敗するテストから。
2. **テストは1本だけ** 赤にする。まとめ書きしない。
3. **緑にする最小の変更だけ** する。先回り機能禁止。
4. 緑になったら **すぐコミット**（メッセージは何が通ったか一言）。
5. リファクタは **全部緑のときだけ**。ついでに仕様追加しない。
6. 迷ったら **中継が通るか**（または今のターゲットの最小テスト）に戻る。
7. ネット必須のテストは書かない（fixture / モック）。無料枠とCIを削らない。
8. 一度に触るパッケージは **できるだけ1つ**（`proxy` か `schema`）。

---

## 1サイクル（コピペ用）

```text
[ ] 次に欲しい振る舞いを日本語1行で書く
[ ] それを表すテストを1本書く → 実行 → 赤を確認
[ ] 本番を最小だけ書く → 実行 → 緑を確認
[ ] git commit -m "test: …" または "feat: … (green)"
[ ] 必要ならリファクタ → 全部緑 → またコミット
[ ] 次の1行へ（戻らない・飛ばさない）
```

---

## 今のプロジェクトでの順番（飛ばさない）

1. 中継が通る（health / 1 completion）— 入口済みなら「回帰テスト1本」
2. RoutePlan が「計画だけ」返す（HTTPなし）
3. Executor が plan の primary を使う
4. 失敗分類つきで fallback 1段
5. CostEstimate 最小（input/output のみ）
6. Offering を設定 or fixture から1本読む

スクレイピング・Web・広告は **脳レスTDDの対象外（後回し）**。

---

## テストの置き場（仮）

| 対象 | 置き場 | 実行 |
|---|---|---|
| schema 型・純関数 | `packages/schema` または将来 `*.test.ts` | 後で test runner |
| proxy の plan / cost | `packages/proxy/src/**/*.test.ts` | 後で test runner |
| 上流HTTP | `fixtures/` + モック。実APIは手動たまに |

**いま test runner が無くても:**  
「期待する入出力をコメントまたは assert ファイルに書く → 実装 → 手で確認」でも脳レスTDDとする。  
runner を足す最初の緑は **「npm test が1本通る」** 自体を1サイクルにしてよい。

---

## コミットメッセージ例

- `test: route plan returns sole offering`
- `feat: make plan green for sole offering`
- `refactor: extract scorer (tests green)`

---

## エージェント向け

- ユーザが脳レスTDD中なら: **次の赤テスト1本** か **緑にする densest パッチ** だけ出す。
- 長い設計講義・横展開・「ついでに」禁止。
- 報告は `docs/USER_STATUS_TEMPLATE.md` の短さで。

---

## 口癖

- 赤がない緑は信用しない
- 1本以上赤のままコミットしない
- 賢くなったら止めろ。サイクルに戻れ
