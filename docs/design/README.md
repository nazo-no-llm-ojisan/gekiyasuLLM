**前提**: 中央は原則としてユーザーの LLM リクエストを中継しない。ルーティング実行は利用者 PC 上のローカル Proxy が担う。

## リポジトリ方針

| 項目 | 内容 |
|---|---|
| 公開 | **最初から GitHub public 前提** |
| 性格 | 設計と個人向け MVP 実装を同一リポに混在してよい |
| 設計 | 本ディレクトリ（日本語） |
| 実装 | 主に `packages/proxy` 等（[AGENTS.md](../../AGENTS.md) の推奨配置） |
| 当面の実装範囲 | Phase 1〜3（個人 Proxy・静的フィード・ローカル統計） |
| 後段 | Phase 4 以降（中央サイト、日次配信、月額、広告）は設計を揃えてから |

### 公開時の表現・免責

- 価格・無料枠・可用性・速度は **保証しない**。`as_of` と情報源を付ける
- 第三者サービスを貶める曖昧語（例: 「怪しい」「グレー」）は使わず、検証可能な属性と trust で表す
- 競合比較は事実ベースの差分整理にとどめる
- 秘密鍵・API キー・実プロンプトをリポジトリに置かない

設計だけ読んで実装を始められること、実装が進んだら設計の「現状 / Phase」を更新することを目標にする。
