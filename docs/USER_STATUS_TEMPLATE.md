# ユーザ向け現状報告テンプレ（脳を使わない版）

報告するとき・自分用メモのとき、**このままコピペして穴埋め**する。  
技術詳細は書かない。長くしない。

## 役割（これだけ覚えてよい）

- **ユーザ:** 大枠判断（やる／やらない、どっち優先、止める）
- **エージェント:** 細かい技術要件を考え、実装し、**docs に残す**
- ユーザは API 名・型・ポートの中身を覚えなくてよい。聞かれて選ぶのは大枠だけ。

詳細開発ループ: [BRAINLESS_TDD.md](./BRAINLESS_TDD.md)

## 口癖

- 迷ったら「中継が通るか」に戻る
- 一度に一つのことだけ
- 「賢く」しない。動いたらコミット
- **脳レスTDD:** 赤 → 緑 → コミット（[BRAINLESS_TDD.md](./BRAINLESS_TDD.md)）
- **並列:** 設計は直列・契約済みは並列（[PARALLEL_AGENTS.md](./PARALLEL_AGENTS.md)）。自分の T-0xx 以外と path を重ねない

## エージェントへの返し方（最短）

次のメッセージはどれか1つでよい:

- `起動コマンドだけ`
- `通った`
- `通らない`（できれば: キー入れた？ / 起動した？ / URLは？）
- `次の一手だけ`

エージェントは **次の一手だけ** 返す。長文の講義をしない。
