# docs/GOVERNANCE.md — ガバナンス・監査・コンプライアンス規約

本リポジトリの公開方針、表現ルール、ライセンス、および法務コンプライアンスに関する規約です。全体監査や、巨大なコンテキストウィンドウ（128k以上など）を持つモデルが確認するために用います。

---

## 1. リポジトリの性格
- **GitHub 公開前提**: 最初から public で開発します。「信用して」ではなく外部から **検査できる** 状態を維持することが大前提です。
- **ライセンス**: Apache-2.0 (`LICENSE`)。名称やロゴは `TRADEMARKS.md` で分離します。
- **企画と MVP 実装の混在**: 設計と個人向け MVP 実装を同一リポジトリに混在させて構いません。実装とドキュメントは並走して更新します。
- **商用・本番利用の制限**: 商用課金の本番運用・有料契約・本番インフラへの接続はメンテナの明示的な承認が必要です。

## 2. 公開リポジトリでの表現ルール
- **価格や可用性の免責**: 価格・無料枠・可用性・速度は一切保証しません。記載する際は必ず情報源 URL と取得日（`as_of`）を付記します。
- **最安・無料の表現**: 「最安」「無料」という言葉を使う場合は、条件付き・時点付きであることが伝わる書き方にします。
- **第三者プロバイダへの配慮**: 第三者プロバイダや仲介 API を誹謗中傷したり、根拠なく「怪しい」等の曖昧な貶め表現を使ったりしないでください。データ利用条件や障害履歴などの観測可能な事実と `trust` フィールドを用いて示します。
- **規約遵守**: 利用規約に反するスクレイピング、過大なプローブ、共有 API キーの濫用を実装・提案しないでください。

## 3. 利益相反 (COI) フィールド
- provider フィードおよび offering には、`sponsored`, `affiliate`, `editorial_rank_influence: "none"` を必須とします。
- 既定のソート順やルーティングのスコアリングに、商業的な関係性に基づくバイアスを介入させてはなりません（編集の分離）。

## 4. 誤記と訂正方針
- 価格情報などの誤記を訂正する際は、`docs/CORRECTIONS.md` に従い、影響期間・原因・フィード版数を記録します（黙って履歴を削除しないでください）。
